HTML

  • The head element contains information about the webpage; it is a container for metadata.
  • Inside head elements, metadata tags can also be used.
  • The title metadata tag defines the title of the webpage.
  • The style element defines CSS styles associated with the webpage.
  • The base element defines the base URL for the webpage.
  • The meta element defines metadata such as: the character set, description, keywords, author, and viewport.
  • The script element embeds client-side scripts in the HTML document.
  • The body element represents the visible content shown to the user.
  • The body of an HTML document contains child and parent elements.
  • Child elements are elements nested inside of other elements.
  • The parent element holds all the child elements.
  • Remember: not all elements are semantic.
  • Semantic HTML elements have names that describe their content, and the role they play on the webpage.
  • The heading element represents the level of heading a given text block represents.
  • Order matters when using heading elements and the biggest heading should only appear once on the page.
  • The paragraph element represents paragraphs, or blocks of text.
  • The list elements including: unordered lists, ordered lists, list items, are used to create lists on the webpage.
  • The image element contains information about images displayed on the webpage.
  • The break element creates a line of empty space between two blocks of content on the webpage.
  • The anchor element creates links to the same webpage or other webpages.
  • The section element defines a section in a document that contains thematic content.
  • HTML attributes define additional information about an element.

CSS

  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.
  • There are three ways to style using CSS: inline CSS, internal CSS style sheet, and external CSS style sheet.
  • The CSS box model dictates that every HTML element is essentially a box with margins.
  • In CSS a wild card symbol * can be used to apply a specific rule to all of the elements visible on a page.
  • Heading text can be styled to be centered on a webpage using the rule property "text-align".
  • The img selector can also be used to apply CSS styling rules to images.
  • A CSS rule must contain a selector, and declaration. The declaration must contain both a property and a value.
  • Within CSS, a selector is the element or attributes that rules (declarations) will apply to.
  • Declarations contain two important components: 1) the CSS property that we want to apply, 2) the value of said property.

Git

  • git status: checks what branch we are currently on
  • git checkout -b branch-name: creates a new branch and switches to it
  • git revert SHA: reverts back to the version of code that is related to the SHA
  • git add -A: adds all modifications in the current working branch to the staging area
  • git commit -m "[description]": commits the staged changes to repo
  • git pull: pulls information from the tracking remote branch and merges it
  • git push: pushes local branch information to the remote repo branch

JavaScript

  • JavaScript is a programming language allows users to interact with websites.
  • JS is case-sensitive, syntax and capitalization matters.
  • A variable is a named container that allows us to store data in our code.
  • Control flow is the order in which a computer executes code in a script.
  • A string is a sequence of text that is enclosed in single or double quotation marks.
  • Number values do not need quotation marks.
  • A Boolean is a true/false value.
  • An array is a structure that allows us to store multiple values in a single reference.
  • Everything in JS is an object and can be stored in a variable.
  • Comments are snippets of text that can be added along with code, which are ignored by the web browser.
  • Operators are mathematical symbols that produce a result based on 2 variables, or values.
  • Conditionals are code structures that are used to test if an expression returns true or not.
  • If...else statements are the most common form of conditionals.
  • In JS the order of the code affects the outcome, because control flow dictates that files will be executed from top to bottom.
  • Conditional statements and loops can be used to interupt control flow, and change the order in which code is executed.
  • An array is a single variable used to hold a group of data, and a single array can store multiple different types of data.
  • A for loop uses the predicatable pattern of indices to perform a task on all the items in an array.
  • A for loop contains three important statements: a keyword, a three-part conditional statement, and a code block.
  • Functions are a set of instructions that tell the computer how to perform a certain task.
  • Functions do not automatically execute when a JS file is run; they must be called in the code at the spot where developers want the function to execute.
